# 美搭 · 已知问题与限制

> 记录当前 Vercel 部署下已知的坑，避免重复排查、误当 bug。多数「serverless 类」问题在迁到国内单机服务器后自动消失（见 `DEPLOY.md` 第七节）。

## 一、访问 / 微信（最影响国内使用）

- **微信内置浏览器里试穿结果被拦 / 出不来**：域名指向 Vercel（海外）且未备案，微信对海外未备案站点限制严格，加上试穿是 10-40 秒的海外长请求，容易被掐断。**治本 = 迁国内服务器 + ICP 备案**，前端优化只能缓解。
- **国内访问 Vercel 慢/不稳**：走海外节点，时快时慢、部分地区/运营商打不开。仅适合给能科学上网的人小范围试。
- 绑了 `www.meidaai.com` 只解决了「裸 vercel.app 打不开」，**没解决**海外+未备案导致的以上两条。

## 二、Serverless 结构性隐患（Vercel 多实例才有，单机服务器无）

当前任务队列/配额用进程内存（未接 Upstash），Vercel 会起多个函数实例、实例间不共享内存、冷启动清空，因此：

- **拆图后台任务可能"丢失"**：提交拆图后前端每 3 秒轮询，若轮询打到**另一个没有该任务的实例** → 返回 missing → 提示「拆分任务已丢失，请重新上传」，但那次生成可能已在原实例跑完、钱已花。低流量下 Vercel 常复用同一热实例，多数能成，但有概率。
- **每日配额不可靠**：拆图 5 次/试穿 20 次的计数散在各实例内存、冷启动清零，实际很容易被绕过。
- **长任务可能被平台掐断**：函数最长 300 秒（`vercel.json` maxDuration），多件拆图叠加限流重试的尾部情况可能超时，未完成的件卡住。
- **首次请求冷启动慢**：一段时间没人访问后，第一个请求（如当天第一次「获取验证码」）要多等几秒，属正常。

> 迁到单机服务器后：单进程、内存共享、任务不丢、配额可靠、无 300 秒上限——以上全消失。

## 三、用户数据

- **只有轻量资料跨设备**：账号、造型档案、偏好、收藏、衣橱选择存 Supabase（换设备/清缓存重登可找回）。**用户上传的衣服照、模特照、试穿结果图仍只在本机 localStorage**，换设备不同步、清缓存会丢。
- localStorage 有 ~5MB 上限，图片超限时会从最老的试衣记录开始自动清理。

## 四、安全余项（评审发现，测试期暂接受，公开前应处理）

已修：`AUTH_SECRET` 线上漏配拒绝启动、登录接口防爆破（每邮箱限次）。**未修**：

- `/api/segment-one`、`/api/segment/retry` 有登录校验但**无配额**：登录用户可循环调用、绕过每日拆图上限烧钱。
- `/api/validate-photo`、`/api/recommend` **无登录校验**：匿名可调用付费模型刷成本。
- **换邮箱可绕过配额**：登录即注册、无真实邮箱门槛前（devCode 模式），换个邮箱就是新配额。接了 DirectMail 后验证码发真邮箱，缓解了一部分。

## 五、成本 / 密钥

- 试穿、拆图、AI 搭配、照片质检**每次都调付费模型**（明细见 `交接文档.md` 花钱须知）。建议在百炼/OpenRouter 后台设**月度消费上限**兜底。
- **上线前待办**：开发过程中在聊天/协作里出现过的密钥（尤其 DashScope、AUTH_SECRET），**建议轮换一遍**再正式对外。

## 六、体验小问题

- 邮箱验证码新发信域名头几封**可能进垃圾箱**（已配 SPF/DKIM/DMARC，会逐步改善）；收不到先翻垃圾邮件。
- 试穿生成图偶发把原图里的电商水印/背景元素"带入"（生图模型固有问题，非本项目 bug）。
