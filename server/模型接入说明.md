# 模型 API 接入说明（给负责接模型的同学）

**当前状态（2026-07 已接真实模型并实测通过）：**

| 能力 | 文件 | 模型 | 平台 | 实测耗时 |
|---|---|---|---|---|
| 照片质检 | ai/validate.js | qwen3.6-flash | OpenRouter | ~5s |
| 识别+平铺图+标签 | ai/segment.js | qwen3-vl-30b + gemini-3.1-flash-image | OpenRouter | ~22s/3件 |
| 试穿生成 | ai/tryon.js | aitryon-plus (OutfitAnyone) | 阿里云百炼 | ~90s |
| 搭配推荐 | ai/recommend.js | qwen3.6-flash（读衣橱标签选搭配，返回搭配理由；失败自动规则兜底） | OpenRouter | ~3s |

- 密钥在 `server/.env`（不进 git）：OPENROUTER_API_KEY、DASHSCOPE_API_KEY
- **换模型只改 `ai/config.js` 里的 MODELS 常量**（平铺图想更精致可换 gpt-5.4-image-2，实测约2.5分钟/张）
- 提示词在 `ai/prompts.js`（与 `ai/提示词库.md` 同步维护）
- 所有接口都有降级：没密钥/调用失败时自动回退占位行为，页面不会坏

## 怎么启动

```bash
cd server
npm install        # 第一次运行执行一次
npm start          # 启动后浏览器打开 http://localhost:8394/login.html
```

手机体验：和电脑连同一个 Wi-Fi，手机浏览器访问 `电脑IP:8394/login.html`。

## 三个待替换的模型接口

| 文件 | 能力 | 现在的占位行为 | 接真模型后 |
|---|---|---|---|
| `server/ai/segment.js` | 衣物抠图 | 原图直接返回 | 返回透明底抠图；一图多件时拆分 |
| `server/ai/tryon.js` | 虚拟试穿 | 返回 null，前端用衣服叠图兜底 | 返回模特上身效果图 |
| `server/ai/recommend.js` | 搭配推荐 | 规则搭配（优先用户上传的衣服） | 模型/大模型推荐 |

每个文件顶部都有详细注释：**输入输出格式（契约）+ 替换示例代码**。
只要保持输入输出格式不变，里面怎么调模型都行。

**各模型的提示词已整理在 `server/ai/提示词库.md`**（含质检/穿着识别/平铺图生成/打标签），
接入时直接复制使用，类别映射表也在里面。

## 接口契约速查

```
POST /api/segment    { image }                    → { image, items:[{image,category}] }
POST /api/tryon      { modelImage, items:[...] }  → { image }        // null=前端叠图兜底
POST /api/recommend  { wardrobe:[...], around }   → { items:[id,...] }
GET  /api/health                                  → { ok: true }
```

## 约定

1. **密钥放环境变量**（如 `SEGMENT_API_KEY`），启动时 `SEGMENT_API_KEY=xxx npm start`，不要写死在代码里、不要提交到 git
2. 图片一律 base64 DataURL 格式传输（`data:image/jpeg;base64,...`）
3. 前端有降级机制：后端挂了或接口报错，页面自动回退本地模拟，不会白屏——所以放心改，改坏了页面也能用
4. 改完一个接口就 git commit 一次（中文说明）
